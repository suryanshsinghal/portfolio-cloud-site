AWS Portfolio Website

S3 + CloudFront + HTTPS Deployment

Problem

When building a personal portfolio website, one of the key challenges is ensuring that it is fast, reliable, secure, and accessible globally.

Traditional hosting methods often lead to performance bottlenecks, higher latency for distant users, and increased operational complexity.

Additionally, security and HTTPS configuration are not straightforward in basic setups, making it difficult to enforce safe and controlled access.

The objective was to build a scalable, secure, and high-performance static website architecture using cloud-native solutions.

Solution

I designed a cloud-based architecture using Amazon S3 for static hosting and CloudFront for global content delivery.

CloudFront caches content at edge locations worldwide, reducing latency and improving load times significantly.

To enhance security, Origin Access Control (OAC) was configured, ensuring that S3 content is only accessible through CloudFront.

HTTPS was enabled for secure communication, and caching strategies were optimized for performance and update efficiency.

Architecture Overview

The system follows a simple and efficient architecture:

  • Users access the website via CloudFront edge locations
  • Cached content is served instantly when available
  • Requests are forwarded to S3 when needed
  • S3 securely stores all static assets

This ensures scalability, low latency, and high availability with minimal operational effort.

Impact

The deployment significantly improved performance, security, and scalability.

  • Reduced latency through CDN distribution
  • Improved user experience globally
  • Enhanced security using OAC and HTTPS
  • Minimal maintenance with serverless architecture

This project demonstrates how modern cloud services can transform a simple static site into a production-grade system.

Learnings

This project provided hands-on experience with cloud architecture and content delivery systems.

  • Understanding CDN and caching mechanisms
  • Implementing secure access controls
  • Managing cache invalidation strategies
  • Designing scalable cloud-native solutions

It strengthened my understanding of real-world cloud deployments and system design.
